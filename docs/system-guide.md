# MarketProbe — 事業検証自動化システム

## 1. システム概要

MarketProbeは、新規事業の市場調査から集客・改善までを自動化するシステムです。
AI（Claude / Gemini）が市場分析・事業案生成・LP作成・SNS投稿・営業・広告運用を自動実行し、
人間は意思決定（承認・却下）に集中できます。

**ダッシュボードURL**: https://lp-app-pi.vercel.app/dashboard

---

## 2. パイプライン（自動処理フロー）

### 2.1 個別パイプライン（ステップ単位実行）

```
市場調査(A) → 市場選定(B) → [人間が承認] → 競合調査(C) → 事業案生成(0) → [人間が承認]
→ LP生成(1) → SNS投稿(2) → フォーム営業(3) → 分析・改善(4) → 広告運用(8)
```

| # | ステップ | 処理内容 | スケジュール | 人間の操作 |
|---|---------|---------|------------|-----------|
| A | 市場調査 | PEST分析・TAM/SAM推定・業界構造・サブレイヤー分解・ペインポイント抽出。証憑（出典情報）付き | 毎週日曜 20:00 | 不要 |
| B | 市場選定 | 5軸スコアリング（歪み深度・参入障壁・BPO適性・成長性・自社適合）。根拠ソース付き | 毎週日曜 22:00 | **承認** |
| C | 競合調査 | 直接/間接/代替競合の分析・ギャップ特定。実在企業のみ（架空企業禁止）、情報源付き | 毎週月曜 4:00 | 不要 |
| 0 | 事業案生成 | 全コンテキスト統合（ナレッジ+市場+痛み+競合+学習+CEO）による事業アイデア生成 | 毎週月曜 6:00 | **承認** |
| 1 | LP生成 | 承認された事業案のランディングページをAIが自動作成 | 毎日 9:00 | 不要 |
| 2 | SNS投稿 | X（Twitter）に事業案の宣伝投稿を自動作成・投稿 | 毎日 10:00 / 18:00 | 不要 |
| 3 | フォーム営業 | ターゲット企業を検索し、問い合わせフォームから自動営業 | 平日 11:00 | 不要 |
| 4 | 分析・改善 | GA4データを取得しAIが改善提案を生成 | 毎日 1:00 | 不要 |
| 5 | Slackレポート | 日次の実行結果サマリーをSlack/Google Chatに自動送信。関連シートへのリンク付き | 毎週月曜 8:00 | 確認のみ |
| 6 | 広告モニタリング | Google Ads予算・入札の自動チェック | 毎時（24/365） | 不要 |
| 7 | 学習エンジン | 実績データからパターン・インサイトを自動学習 | 毎日 3:00 | 不要 |
| 8 | 広告自動出稿 | Google Adsキャンペーンの自動作成・最適化 | 毎週月曜 10:00 | 不要 |

### 2.2 自律型パイプライン（orchestrate_abc0）

A〜0のステップに加え、痛み抽出・ユニットエコノミクス・チェックリスト評価・インタビュースクリプト生成・自己反省を**完全無人で一気通貫実行**するモードです。

```
Step A: 市場調査（+ サブレイヤー分解）
  ↓
Step P: 痛み抽出（ナレッジベースから）
  ↓
Step B: 市場選定（痛みデータ反映したスコアリング）
  ↓
自動承認（上位N件を自動で selected に）
  ↓
Step C: 競合調査
  ↓
Step 0: 事業案生成
  ↓
Step U: ユニットエコノミクス算出（LTV/CAC/BEP）
  ↓
Step E: チェックリスト評価（84項目×2書籍＝164項目基準）
  ↓
Step I: インタビュースクリプト生成（30分構成）
  ↓
自己反省 → 改善提案の蓄積 → 次回実行に自動フィードバック
```

**実行方法**:
- ダッシュボード: パイプラインタブの「orchestrate_abc0」の「▶ 実行」ボタン
- Google Chat: 「自律パイプライン実行して」「abc0回して」「パイプライン全部やって」
- CLI: `SCRIPT_NAME=orchestrate_abc0 python run.py`

**自動承認の設定**:
| 設定キー | デフォルト | 説明 |
|---------|-----------|------|
| `orchestrator_auto_approve` | `true` | 市場を自動承認するか |
| `orchestrator_auto_approve_n` | (空=selection_top_n) | 上位何件を自動承認 |
| `orchestrator_min_score_threshold` | `0` | 自動承認の最低スコア |

**自己反省・フィードバックループ**:
パイプライン完了後、AIが全ステップの結果を自己評価します。
- `improvement_suggestions`（改善提案）、`risks_identified`（リスク）、`next_actions`（次回アクション）を生成
- `pipeline_improvement_log`（設定シート）に最新5件を蓄積
- 次回パイプライン実行時にナレッジコンテキストとして自動注入
- 自己反省テンプレートが前回の改善提案をフォローアップ

## 5. ダッシュボードの使い方

### 5.1 パイプラインタブ（メイン画面）

全ステップの状態を一覧で確認できます。

**ステータス表示**:
- 🟢 **Done** — 正常完了
- 🔵 **Running** — 実行中（点滅アニメーション）
- 🔴 **Error** — エラー発生
- ⚪ **Idle** — 未実行

**手動実行**: 各ステップの「▶ 実行」ボタンで即座に手動実行できます。

**スケジューラー制御**: 各ステップのON/OFFトグルで自動実行を一時停止・再開できます。

### 5.2 承認フロー

2つのステップで人間の承認が必要です（自律型パイプラインでは自動承認も可能）:

#### 市場選定の承認
ダッシュボード画面上部の青いバナーに市場名・スコア・推奨参入角度が表示されます。「承認」「却下」ボタンで操作します。

#### 事業案の承認
事業案生成後、同様にダッシュボード/Slackで承認が必要です。

### 5.3 知識ベースタブ

PDFの書籍やドキュメントをアップロードし、AIの分析精度を向上させます。

**アップロード方法**:
1. 「知識ベース」タブをクリック
2. PDF をドラッグ&ドロップ
3. AIが自動で書籍の要約・章構成・フレームワークを抽出

**チャット機能**: アップロードした書籍の知識を使ってAIに事業相談ができます。

### 5.4 ログタブ

実行履歴を時系列で確認できます。

## 6. データ管理

全データはGoogle Spreadsheet「事業検証自動化ログ」に保存されます。
通知メッセージ内のリンクをクリックすると、該当シートタブを直接開けます。

### 6.1 主要シート

| シート名 | 内容 | 確認すべきタイミング |
|---------|------|-------------------|
| **market_research** | 市場調査結果（PEST分析・TAM/SAM・サブレイヤー・証憑） | パイプライン実行後に調査結果を確認 |
| **market_selection** | 5軸スコアリング結果・承認ステータス・根拠ソース | 承認/却下の判断時 |
| **competitor_analysis** | 競合調査結果・ギャップ分析・情報源 | 事業案の差別化戦略検討時 |
| **business_ideas** | 事業アイデア・UE分析・チェックリスト評価 | 事業案の承認/検討時 |
| **settings** | 全設定値（市場・重み・方向性メモ・改善ログ等） | 方針変更時 |

### 6.2 運用シート

| シート名 | 内容 |
|---------|------|
| **pipeline_status** | 各ステップの最新実行ステータス |
| **execution_logs** | 実行ログ（手動実行の記録含む） |
| **knowledge_base** | アップロード済みPDF書籍の情報 |
| **learning_memory** | AI学習メモリ（パターン・インサイト・指示） |
| **lp_content** | 生成されたLPコンテンツ |
| **sns_posts** | SNS投稿履歴 |
| **form_sales_targets** | フォーム営業の送信記録 |
| **analytics** | GA4から取得したアクセス分析データ |
| **improvement_suggestions** | AIが生成したLP改善提案 |
| **performance_log** | パフォーマンス集計記録 |
| **ads_campaigns** | 広告キャンペーン情報 |

### 6.3 証憑（エビデンス）

市場調査・市場選定・競合調査の出力には情報源が付与されます:

- **market_research**: `data_sources`カラムにJSON配列で出典情報（調査レポート名、政府統計、IR資料等）。`confidence_score`（1-5）と `confidence_rationale` で信頼度を明示
- **market_selection**: `rationale_sources`カラムにスコアリング根拠の情報源リスト
- **competitor_analysis**: `info_source`カラムに各競合の情報源（公式サイト/IR資料/業界レポート/推定）。架空の企業名・URLは禁止

## 8. システム構成

### 技術スタック

| レイヤー | 技術 | 用途 |
|---------|------|------|
| フロントエンド | Next.js 16 + Tailwind CSS | ダッシュボードUI |
| ホスティング | Vercel | ダッシュボード配信 |
| AI（主系） | Gemini 2.5 Flash | テキスト分析・生成（コスト効率優先） |
| AI（副系） | Claude Sonnet 4.5 | Gemini障害時の自動フォールバック |
| AI（チャット） | Gemini 2.5 Pro | Google Chatボットの対話 |
| ジョブ実行 | Google Cloud Run Jobs | パイプラインスクリプト実行 |
| スケジューリング | Google Cloud Scheduler | 定期実行管理 |
| データ保存 | Google Sheets | 全データの一元管理 |
| ファイル保存 | Google Cloud Storage | PDF書籍の保存 |
| 通知 | Slack Webhook + Google Chat Webhook | 同時通知 |
| チャットボット | Google Chat API (Pub/Sub) | 対話型操作 |
| SNS | X (Twitter) API | 自動投稿 |

### 可用性

- **AIフォールバック**: Gemini APIが使用不能の場合、自動でClaude APIに切り替え
- **リトライ**: JSON生成失敗時は最大3回リトライ（温度を段階的に下げて安定化）
- **バリデーション**: 各ステップの出力をJSON構造・スコア範囲・必須フィールドで検証
- **エラー通知**: エラー発生時はSlack/Google Chatに即座に通知

## 9. よくある操作

### 自律型パイプラインを実行する
- Google Chat: 「自律パイプライン実行して」
- ダッシュボード: orchestrate_abc0 の「▶ 実行」
- 所要時間: 約20-30分

### 調査対象の市場を変更する
- Google Chat: 「探索市場をエネルギー,医療,物流に変更して」
- ダッシュボード: 設定ページで `exploration_markets` を変更

### 方向性メモを設定する
- Google Chat: 「BtoB SaaSを重視する方向で調査して」
- → `market_direction_notes` に自動保存され、次回調査で考慮される

### 通知のシートリンクからデータを確認する
通知メッセージ内の「📊market_research」などのリンクをクリックすると、スプレッドシートの該当タブが直接開きます。

### エラーが出た場合
1. Slack/Google Chat通知でエラー内容を確認
2. 通知内のシートリンクでデータ状態を確認
3. ダッシュボードの「▶ 実行」で再実行、またはGoogle Chatから「〇〇を実行して」
