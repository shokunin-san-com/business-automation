# MarketProbe — 事業検証自動化システム

## 1. システム概要

MarketProbeは、新規事業の市場調査から集客・改善までを自動化するシステムです。
AI（Claude / Gemini）が市場分析・事業案生成・LP作成・SNS投稿・営業・広告運用を自動実行し、
人間は意思決定（承認・却下）に集中できます。

**ダッシュボードURL**: https://lp-app-smoky.vercel.app/dashboard

---

## 2. パイプライン（自動処理フロー）

### 全体の流れ

```
市場調査 → 市場選定 → [人間が承認] → 競合調査 → 事業案生成 → [人間が承認]
→ LP生成 → SNS投稿 → フォーム営業 → 分析・改善 → 広告運用
```

### 各ステップ詳細

| # | ステップ | 処理内容 | スケジュール | 人間の操作 |
|---|---------|---------|------------|-----------|
| A | 市場調査 | 対象市場のPEST分析・TAM/SAM推定・業界構造・ペインポイント抽出 | 毎週日曜 20:00 | 不要 |
| B | 市場選定 | 5軸スコアリング（歪み深度・参入障壁・BPO適性・成長性・自社適合）で市場をランク付け | 毎週日曜 22:00 | **Slack/ダッシュボードで承認** |
| C | 競合調査 | 承認された市場の競合分析・ギャップ特定・参入機会の発見 | 毎週月曜 4:00 | 不要 |
| 0 | 事業案生成 | 市場調査・競合分析の結果を踏まえた事業アイデアを自動生成 | 毎週月曜 6:00 | **Slack/ダッシュボードで承認** |
| 1 | LP生成 | 承認された事業案のランディングページをAIが自動作成 | 毎日 9:00 | 不要 |
| 2 | SNS投稿 | X（Twitter）に事業案の宣伝投稿を自動作成・投稿 | 毎日 10:00 / 18:00 | 不要 |
| 3 | フォーム営業 | ターゲット企業を検索し、問い合わせフォームから自動営業 | 平日 11:00 | 不要 |
| 4 | 分析・改善 | GA4データを取得しAIが改善提案を生成 | 毎日 1:00 | 不要 |
| 5 | Slackレポート | 週次の実行結果サマリーをSlackに自動送信 | 毎週月曜 8:00 | 確認のみ |
| 6 | 広告モニタリング | Google Ads予算・入札の自動チェック | 毎時（24/365） | 不要 |

## 3. ダッシュボードの使い方

### 3.1 ログイン

ダッシュボードURL（https://lp-app-smoky.vercel.app/dashboard）にアクセスします。

### 3.2 パイプラインタブ（メイン画面）

パイプラインタブでは、全10ステップの状態を一覧で確認できます。

**ステータス表示**:
- 🟢 **Done** — 正常完了（最終実行が成功）
- 🔵 **Running** — 実行中（点滅アニメーション）
- 🔴 **Error** — エラー発生
- ⚪ **Idle** — 未実行

**手動実行**: 各ステップの「▶ 実行」ボタンで即座に手動実行できます。

**スケジューラー制御**: 各ステップのON/OFFトグルで自動実行を一時停止・再開できます。
画面上部の「全体開始/全体停止」で一括制御も可能です。

### 3.3 承認フロー

2つのステップで人間の承認が必要です：

#### 市場選定の承認
市場選定（B）の実行後、スコアの高い市場がSlackとダッシュボードに表示されます。

- **ダッシュボード**: 画面上部の青いバナーに市場名・スコア・推奨参入角度が表示されます。「承認」「却下」ボタンで操作します。
- **Slack**: 通知が届きます。

承認した市場のみ、次の競合調査（C）で分析されます。

#### 事業案の承認
事業案生成（0）の実行後、同様にSlack/ダッシュボードで承認が必要です。
承認された事業案のみ、LP生成・SNS投稿・フォーム営業の対象になります。

### 3.4 知識ベースタブ

PDFの書籍やドキュメントをアップロードし、AIの分析精度を向上させます。

**アップロード方法**:
1. 「知識ベース」タブをクリック
2. PDF をドラッグ&ドロップ、または「ファイルを選択」をクリック
3. AIが自動で書籍の要約・章構成・フレームワークを抽出

**チャット機能**:
アップロードした書籍の知識を使って、AIに事業相談ができます。
チャット欄に質問を入力すると、書籍の内容に基づいた回答がストリーミングで表示されます。

### 3.5 ログタブ

実行履歴を時系列で確認できます。エラー発生時はここで原因を確認します。

## 4. Slack通知

パイプラインの状態変化がSlackに自動通知されます。

| 通知タイミング | 例 |
|--------------|---|
| ステップ開始 | 🔍 市場調査 ▶️ 開始 |
| ステップ完了 | 💡 事業案生成 ✅ 完了 → 3件生成 |
| エラー発生 | ❌ LP生成 ❌ エラー → API timeout |
| 承認リクエスト | 💡 新しい事業案: ○○ — 承認してください |

## 5. データ管理

全データはGoogle Spreadsheet「事業検証自動化ログ」に保存されます。

| シート名 | 内容 |
|---------|------|
| settings | 調査対象市場・スコアリング重みなどの設定 |
| market_research | 市場調査結果（PEST分析・TAM/SAM・ペインポイント） |
| market_selection | 市場スコアリング結果・承認ステータス |
| competitor_analysis | 競合調査結果・ギャップ分析 |
| business_ideas | 生成された事業アイデア |
| lp_content | 生成されたLPのHTMLコンテンツ |
| sns_posts | SNS投稿履歴 |
| form_submissions | フォーム営業の送信記録 |
| analytics_suggestions | AI改善提案 |
| pipeline_status | 各ステップの実行ステータス |
| execution_logs | 実行ログ |
| knowledge_base | アップロード済みPDF書籍の情報 |
| settings | システム設定値 |

## 6. 設定変更

ダッシュボードの「設定」ページ（/settings）から主要な設定を変更できます。

| 設定項目 | 説明 | デフォルト値 |
|---------|------|------------|
| exploration_markets | 調査対象の市場リスト | エネルギー,IT,建設,製造,物流 |
| exploration_segments_per_market | 市場あたりの調査セグメント数 | 3 |
| selection_top_n | 承認候補として提示する上位市場数 | 3 |
| competitors_per_market | 市場あたりの競合分析数 | 5 |
| exploration_scoring_weights | 5軸スコアリングの重み配分 | 各軸均等 |

## 7. システム構成

### 技術スタック

| レイヤー | 技術 | 用途 |
|---------|------|------|
| フロントエンド | Next.js 16 + Tailwind CSS | ダッシュボードUI |
| ホスティング | Vercel | ダッシュボード配信 |
| AI（主系） | Claude Sonnet 4.5 | テキスト分析・生成 |
| AI（副系） | Gemini 2.5 Flash | Claude障害時の自動フォールバック |
| ジョブ実行 | Google Cloud Run Jobs | パイプラインスクリプト実行 |
| スケジューリング | Google Cloud Scheduler | 定期実行管理 |
| データ保存 | Google Sheets | 全データの一元管理 |
| ファイル保存 | Google Cloud Storage | PDF書籍の保存 |
| 通知 | Slack Webhook | ステータス通知・承認リクエスト |
| SNS | X (Twitter) API | 自動投稿 |

### 可用性

- **AIフォールバック**: Claude APIが使用不能になった場合、自動でGemini APIに切り替わります。パイプラインは停止しません。
- **リトライ**: Cloud Run Jobsは失敗時に1回自動リトライします。
- **エラー通知**: エラー発生時はSlackに即座に通知されるため、迅速に対応できます。

## 8. よくある操作

### パイプラインを一括で開始する
ダッシュボードの「全体開始」ボタンをクリック → 全スケジューラーがONになり、設定時刻に自動実行が始まります。

### 特定のステップだけ今すぐ実行する
該当ステップの「▶ 実行」ボタンをクリック → Cloud Run上で即座に実行されます。

### 調査対象の市場を変更する
設定ページ（/settings）で `exploration_markets` の値を変更します。
例: `エネルギー,IT,建設,製造,物流` → `医療,教育,不動産,農業,金融`

### エラーが出た場合
1. Slack通知でエラー内容を確認
2. ダッシュボードのログタブで詳細を確認
3. 「▶ 実行」で再実行

### パイプラインを全停止する
ダッシュボードの「全体停止」ボタンをクリック → 全スケジューラーがOFFになります。
実行中のジョブは完了まで走りますが、次回以降の自動実行は停止します。
